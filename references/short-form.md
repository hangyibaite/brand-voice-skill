# Short-form Schema

When generating the personalized skill's `references/short-form.md`, populate this exact schema from analysis. Every field must be filled. Empty fields trigger a targeted question to the user.

## Schema to populate

```
# Short-form Voice Rules

## Vocabulary
- Use these (the fingerprints): [list 8–15 words/phrases]
- Cussing: [allowed words + frequency rule, or "none"]
- Avoid: [banned phrases]

## Sentence rhythm
- Average sentence length: [range in words]
- Fragments: [usage — short-form usually heavy]
- Punctuation rules:
  - Em-dashes: [banned / allowed / rare]
  - All caps: [usage]
  - Ellipses: [usage]
  - Parentheticals: [usage]

## Structure
- Hook patterns: [list 2–3 hook styles the user uses + one real example each from samples]
- Open loop placement: [where in the script the loop opens — describe + example]
- Rehook usage: [yes / no / when]
- Closing patterns: [list with examples]
- CTA style: [describe + 2 real examples]

## Stage directions and format conventions
- Bracket conventions: [the exact bracket types the user uses, e.g. [HOOK], [CUTSCENE:], (beat), [CUT]]
- Pace markers: [the user's notation for cuts, beats, pauses]
- Visual / sound cue conventions: [if any]

## POV + tone
- Person: [first / second]
- Relationship to camera/viewer: [one sentence]
- Register: [describe with one example pulled from samples]

## Length default
- Word count: [range, e.g. 120–150 words for 45–60s reel]

## Example excerpts
[Paste 2–3 full short-form pieces from the user's samples. These anchor the voice. The writer should reread these before drafting any short-form piece.]
```

## Reminder

Fill every field using only evidence from the samples and live-write. Do not invent. Quote real examples.
