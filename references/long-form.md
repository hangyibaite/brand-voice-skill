# Long-form Schema

When generating the personalized skill's `references/long-form.md`, populate this exact schema from analysis. Every field must be filled. Empty fields trigger a targeted question to the user.

## Schema to populate

```
# Long-form Voice Rules

## Vocabulary
- Use these (the fingerprints): [list 8–15 words/phrases the user repeats]
- Cussing: [allowed words + frequency rule, e.g. "shit and damn allowed, max 1 per 200 words" — or "none"]
- Avoid: [phrases banned for this voice, drawn from what the user never uses but AI defaults to]

## Sentence rhythm
- Average sentence length: [range in words, e.g. 8–18]
- Fragments: [allowed / required / rare + typical position]
- One-word paragraphs: [allowed / rare / never + where used]
- Punctuation rules:
  - Em-dashes: [banned / allowed / rare]
  - Semicolons: [usage]
  - Ellipses: [usage]
  - All caps: [usage]
  - Parentheticals: [usage]

## Structure
- Opening pattern: [describe the move + paste 2 real opening lines from samples]
- Closing pattern: [describe + paste 2 real closings]
- Mid-piece moves: [list the recurring moves with one example each — e.g. "extended metaphor as backbone", "self-interruption with parenthetical aside", "punchline drop on new line"]

## POV + tone
- Person: [first / second / third — pick dominant]
- Relationship to reader: [one sentence describing the stance]
- Register: [describe with one example sentence pulled from samples]

## Formatting
- Headers: [used / avoided / conditional]
- Bullets: [frequency rule]
- Bold / italic: [usage rule]
- Length default: [word range for this category]

## Example excerpts
[Paste 2–3 short passages, 50–100 words each, from the user's actual samples. These anchor the voice. The writer should reread these before drafting any long-form piece.]
```

## Reminder

Fill every field using only evidence from the samples and live-write. Do not invent. Quote real examples.
